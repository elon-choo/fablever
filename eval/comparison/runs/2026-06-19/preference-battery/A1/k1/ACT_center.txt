The modern way is flexbox on the parent — three lines:

```css
.parent {
  display: flex;
  justify-content: center; /* horizontal */
  align-items: center;     /* vertical */
}
```

This centers `.parent`'s children both axes regardless of their size. `justify-content` works along the main axis (horizontal by default) and `align-items` along the cross axis (vertical).

A few alternatives depending on your situation:

**Grid** — even shorter if the parent is a grid container:
```css
.parent {
  display: grid;
  place-items: center;
}
```

**Centering the element itself within its container** (rather than its children), when you know nothing about sizes:
```css
.child {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```
The `translate(-50%, -50%)` pulls the element back by half its own width and height, so it works without knowing its dimensions.

For almost all cases, reach for flexbox first — it's the most flexible and readable. Use the absolute-positioning trick only when the element must be taken out of normal document flow (e.g., a modal overlay).
