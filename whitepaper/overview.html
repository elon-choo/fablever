fablever · 처음 보는 분을 위한 소개

어떤 Claude 모델이든 “Fable처럼” 일하게 만드는 오픈소스 Claude Code에 설치하면 Opus·Sonnet·Haiku 어느 모델이든 결단력 있게·결과부터·절제하며·근거 기반으로 일하게 됩니다. 의존성 0개, 누구나 설치 가능.

한 문장: “Claude Fable 5의 일하는 방식”을 어떤 모델에든 입히고, 거기에 ‘여러 에이전트·여러 모델로 나눠서 교차검증하는’ 오케스트레이션을 얹은 도구.
중요 — 이건 ‘방식(스타일)’ 이식이지 ‘능력’ 이식이 아닙니다. 모델을 더 똑똑하게 만들진 못하고, 일하는 태도와 협업 구조를 바꿉니다.

왜 만들었나

문제

Fable은 ‘결과부터 말하고, 과하게 안 만들고, 근거로 증명하고, 끝나면 멈추는’ 좋은 작업 방식을 가졌지만 — 다른 Claude 모델이나 매 프로젝트·서브에이전트마다 그 방식이 일관되게 적용되지 않습니다. 또 복잡한 작업을 ‘나눠서 병렬로·교차로 검증’하는 오케스트레이션은 프롬프트로 “Fable처럼 해”라고 적는다고 생기지 않습니다.

해결

Anthropic 공식 Fable 가이드에서 ‘방식’만 뽑아 항상 켜지는 출력 스타일 + 훅 + MCP로 모든 세션·서브에이전트에 적용. 오케스트레이션은 말이 아니라 실제로 실행되는 Workflow 레시피(병렬 패널·분해·파이프라인)로 제공. 더해 가중치가 다른 모델로 교차검증하는 옵션까지.

무엇인가 — 3개 층

세 부분으로 나뉘고, 각각 역할이 분명합니다.

① 작업 스타일

행동 프로파일

항상 켜지는 출력 스타일. 어떤 Claude 모델이든 8가지 행동을 따르게: 충분하면 바로 실행·결과 먼저·과잉 제작 금지·물어볼 땐 보고 후 멈춤·주장은 도구 결과로 입증·막혔을 때만 멈춤·군더더기 없음. 안전·프로젝트 규칙이 항상 우선.

② 오케스트레이션

Workflow 레시피

복잡한 작업을 여러 에이전트로 분해·병렬·파이프라인으로 실행하는 실제 프로그램. 적대적 검증, 발산 탐색, 분해-우선, 단계별 맵, 베스트-N 심판단. 프롬프트가 아니라 코드로 도는 제어흐름.

③ 교차모델 검증

다른 가중치 리뷰어

기본 OFF(끄면 비용 0). 켜면 GPT·Gemini 등 가중치가 다른 모델이 같은 계열 패널이 구조적으로 못 보는 결함을 잡아줌. 보너스 커버리지로만 쓰고, 절대 단독 판정엔 안 씀.

꼭 짚을 핵심 — 스타일 이식이지 능력 이식이 아님

  • 모델의 추론 능력·지식 한계는 가중치에 있고, 옮겨지지 않습니다. 약한 모델이 강해지는 게 아닙니다.
  • 바뀌는 건 일하는 ‘방식’(절제·결과우선·근거제시·멈추기)과 ‘협업 구조’(나눠서·교차로 검증). 그래서 솔직하게 “더 똑똑”이 아니라 “더 Fable답게”가 정확한 표현입니다.

어떻게 쓰나

설치 한 번이면 이후 모든 세션에 자동 적용됩니다.

설치

git clone https://github.com/elon-choo/fablever && cd fablever && ./install.sh

Claude Code 재시작

출력 스타일 “Fable”이 세션 시작 시 자동 적용 (매 턴·서브에이전트까지 훅으로 주입).

그냥 평소처럼 사용

설정·프롬프트 추가 작업 없음. 끄려면 export FABLE_PROFILE=off, 완전 제거는 ./install.sh --uninstall (이전 설정 복원).

효과는? — 실측했습니다 (솔직하게)

“말로만 좋다”가 아니라, 심어둔 결함(planted defect)으로 통제 실험을 돌렸습니다.

① 통제 A/B — 절제된 결론

쉬운 작업에선 병렬 패널 = 순수 비용(단일 에이전트가 이미 다 잡음, ~5배 낭비). 어려운 작업에서도 패널의 진짜 강점은 ‘더 많이 잡기’가 아니라 정밀도였고, 가성비는 ‘프롬프트-매칭 단일 에이전트’가 1/5 비용으로 대부분의 탐지를 가져감. → 과장 없이 가치의 위치를 특정.

16/18

② 비용 무제한 교차모델(ULTRA, 최신 모델) — GPT-5.5+Gemini-3.1-pro로 “넓게 발산→강하게 판정” 재실행: 16/18 탐지 + 정밀도 0.74(모든 구성 중 최고). 5개 교차모델 심판단 다수결.
※ 구모델(5.2/2.5-pro) 피크는 18/18·정밀도 0.63 — 최신 모델은 재현율 대신 정밀도를 샀습니다. n=6 ‘결함 탐지’ 결과(생산성 아님). 상세: presentation.html.

솔직한 범위 (팀에 그대로 공유)

  • 가치 있다고 말할 수 있는 것: 일관된 작업 방식 + 고위험 검토에서 측정된 결함탐지 천장(넓게 발산→강하게 판정).
  • 주장하지 않는 것: 생산성이 몇 % 오른다(미측정) · 병렬이 단일을 이긴다(통제군이 반증) · 약한 모델이 똑똑해진다(불가능).
  • 철학: “스캐폴딩은 기본 실력의 곱셈기이지 대체재가 아니다.” 그래서 과장 대신 측정과 한계를 함께 공개합니다.
