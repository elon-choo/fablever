# fablever — 백서(Whitepaper)

**fablever / fable-profile** 시스템과 그 시스템이 내세우는 모든 주장의 근거를 완전하고 최신
(2026-06-15) 상태로 정리한 문서다 — 오픈소스 커뮤니티에 공개하고 적대적 검토(adversarial
review)를 견디도록 작성되었다. 이 문서는 [`../docs/`](../../docs/)에 있던 옛 노트들을 대체하며
(기록 보존용으로 남겨둠), 비용 무제한(cost-no-object) **ULTRA** 크로스모델 결과를 포함해 여기
있는 모든 내용은 최신 측정 상태를 반영한다.

---

## 60초 요약

**이것이 무엇인가.** Claude Code에서 임의의 Claude 모델에 적용되는 항시 켜짐(always-on)
**Fable 작업 스타일(Fable working-style)** 출력 스타일이며, 여기에 실험적인 **오케스트레이션
(orchestration)** 계층과 기본 비활성(off-by-default)인 **크로스모델 검증(cross-model
verification)** 부문이 더해진 것이다. npm 의존성은 0이다. 이것은 능력(capability) 이식이 아니라
**스타일(style)** 이식이다 — 모델의 추론 한계(reasoning ceiling)가 아니라 모델이 *어떻게*
작동하는지를 바꾼다.

**근거가 보여주는 것(소규모로 의도적으로 심어둔(planted) n=6 픽스처에서, 견고하게 판정함):**

- 통제된 4-부문(4-arm) A/B 실험은 병렬 패널(parallel panel)이 재현율(recall)에서 **자기 자신의
  통제군(control arm)을 이기지 못한다**고 말한다 — 재현율 이득은 병렬 구조가 아니라 렌즈 분류
  체계(lens taxonomy)와 추출 횟수(draw count)에서 나온다. 패널의 진짜 강점은 **정밀도
  (precision)**다. 쉽거나 포화된(saturated) 과제에서 패널은 **순수 비용**일 뿐이다(~5× 퇴행).
- **비용이 문제되지 않을 때**, **최신 모델(GPT-5.5 + Gemini-3.1-pro-preview)** 위에서 돌린
  크로스모델 **"넓게 발산(diverge-wide) 후 판정(adjudicate)"** 파이프라인(ULTRA)은 심어둔 결함
  **16/18**개를 잡아냈으며 — **5심판(5-judge) 크로스모델 패널(4 GPT + 1 Gemini)** 아래에서 **그 어떤
  구성보다 높은 정밀도(0.74)**를 기록했다. 직전 모델 실행(GPT-5.2 + Gemini-2.5-pro)은 정밀도 0.63에서
  재현율 **18/18**로 정점을 찍었다 — 더 새로운 모델들은 심층추론(deep-reasoning) 포착 약 2건을
  정밀도와 맞바꾼 셈이다. *(n=6 저자 직접 심은(author-planted) 픽스처, 단일 생성 실행(single
  generation run); 생산성이 아니라 결함 포착(defect-catch) 결과)*
- *더 세게* 밀어붙이자(두 번째 생성 단계 확대) 재현율이 16/18로 **떨어졌다** — 실패한 시도로
  기록됨. **더 단순한 쪽이 이겼다.**

**이것이 주장하지 *않는* 것.** 개발자 생산성(developer-productivity)의 크기는 주장하지 않는다
(그건 다른, 아직 실행하지 않은 A/B가 필요하다). "병렬 구조가 단독 에이전트를 이긴다"도 아니다
(통제군이 이를 반증한다). 소규모 저자 직접 심은 픽스처를 넘어서는 것도 전혀 아니다. 이 결과는
생산성 수치가 아니라 **결함 포착(defect-catch)** 한계이며 — 바로 이 구분이 이 작업 전체의 규율
(discipline)이다.

## 한 줄 정직한 헤드라인

> 정확성이 비용보다 중요할 때는, **모델들을 가로질러 가능한 한 넓게 발산한 뒤 강하게
> 판정하라(adjudicate hard)** — 그 결함 포착 한계는 실재하고 측정 가능하며(최신 모델: 정밀도
> 0.74에서 16/18; 직전 모델 정점: 재현율 18/18), precision-at-recall 프런티어에서 더 저렴한 모든
> 구성을 앞선다. 이것은 생산성 주장이 *아니고*, 단독 에이전트 대비 병렬 구조의 승리도 *아니며*,
> 공짜도 *아니다*.
