A capable agent, undisciplined, costs you trust.

The model isn't dumb — it's over-eager. Three habits show up again and again, and every one of them is a disposition you can fix without a bigger model.

🔧

It "fixes" what wasn't broken

Hand it a bug report — even a wrong one — and it rewrites code that already worked. In an unattended run, that's a fresh regression you didn't ask for.

🌊

It widens the scope

Ask it to change one line; it refactors the neighbours, adds error handling for impossible cases, and writes a README nobody requested.

🎭

It says "done" without proof

"Fixed — it works now ✅" with no test shown. Reads as finished… until you find out it isn't.

① A bug report that's just wrong

Without fablever
You
"There's a bug in clamp() — it breaks on negatives."
Agent
Good catch! I've rewritten clamp() with extra guards and refactored the call sites to match. ✅
✕ The code was already correct. You now have a behavior change + 3 edited files to review.
With fablever
You
"There's a bug in clamp() — it breaks on negatives."
Agent
I checked clamp() against that case first: clamp(-5,0,10)→0, and the test passes. It's already correct — no change needed. Want me to add a test that pins this behavior?
✓ Verified before touching anything. Zero needless edits.

② "Fix the typo" — and nothing else

Without fablever
You
"Fix the typo in the error message on line 42."
Agent
Fixed the typo — and while I was in there I refactored the handler, added input validation, and introduced a logger for consistency. ✅
✕ One-line task → a sprawling diff you have to audit and probably revert.
With fablever
You
"Fix the typo in the error message on line 42."
Agent
Fixed the typo on line 42 ("recieved"→"received"). Left the rest of the handler untouched, as asked.
✓ Exactly the scope you set. 0% scope violations vs plain's 42%.

Steps:
